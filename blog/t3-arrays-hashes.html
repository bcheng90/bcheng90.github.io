Hash(tag) Arrays

#array #hashes #dbc #blog January 11th, 2015

When you receive a ton of data, you'll need a way to sort through it all. The collections of data that you have might also need to be sorted by length or alphabetically. How does one create an index of information like this? With Ruby, it's simple! Kind of. Let's talk about arrays and hashes!

An array is a list of items in order. That's the most simplified answer I can give you. Now to build on this, let's make a list of items:

    Cookie Brands

  • Oreos
  • Chips Ahoy
  • Famous Amos
  • Milano

From here, we now have four values. Each of these values are tied to a key (or in this case, an integer). If written out, the array will look like this:

  • cookie_brands=['Oreos', 'Chips Ahoy', 'Famous Amos', 'Milano']

The "Oreos" value is tied to the key [0]. The "Chips Ahoy" value is tied to [1]. This continues for all other values. It is important to remember that the keys within arrays all begin with 0, rather than 1. So what happens when you want to call on the 3rd value within an array? Look at the following example:

  • cookie_brands[2] = "Famous Amos"

Alright, lets check out Hashes. Hashes are lists in no particular order. They are another form of keys and value pairs. They differ from arrays by allowing the keys to be any object within Ruby. That means a key can be represented by an integer, a string, or a character. It's also to be noted that they're not ordered. Arrays are ordered from 0 to whatever number, but Hashes can be in any order. Let's check out an example.

    Pet Names

  • Turtle:"Franklin"
  • Cat:"Garfield"
  • Rabbit:"Bugs"
  • Dog:"Pluto"

The keys within this list would be the types of pets. The values would be their names. If written into code, it would look like this:

  • pet_names = {"Turtle"=>"Franklin", "Cat"=>"Garfield", "Rabbit"=>"Bugs", "Dog"=>"Pluto"}

Think of hashes kind of like a dictionary. Each word within the dictionary represents a key and each definition would be it's value. If we wanted to call upon the value of rabbit, it'd look like this:

  • pet_names[rabbit] = "Bugs"

Hope this has been helpful in figuring out the differences between arrays and hashes!
