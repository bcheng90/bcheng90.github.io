Classes in Ruby

I'm not talking about DBC Classes! January 24th, 2015

Classes are quickly becoming my main focus point within Phase 0. A Class is simply like a foundation to create an object. In other words, lets say you're a counterfeiter. You have your templates (Class) that creates all your fake dollar bills (objects). Each counterfeit bill, whether it be a single dollar bill, a five dollar bill, a ten, or a twenty are your objects of the templates. The template, which is your class, will create all your bills, even if they don't look all the same at the end result. Let's check this out:

class Counterfeit
 def initialize(bills)
  @bills = bills
 end
 def printing
  if @bills == 1
    p "Made a $1 bill"
  elsif @bills == 5
    p "Made a $5 bill"
  elsif @bills == 10
    p "Made a $10 bill"
  elsif @bills == 20
    p "Made a $20 bill"
  else
    p "Now that's just a super fake bill."
  end
 end
end


new_bill = Counterfeit.new(5)
new_bill.printing

What this is doing here is creating the class of "Counterfeit", initializing the "bills" to an instance variable of "bills". This allows the variable of bills to be used solely within the Class. Defining the method of printing will allow for us to later run the code that follows the "if". This next block of code will only print out the result of "Made a $x bill" where x is equal to the values seen in the code. So at the bottom of it, new_bill is the new object we are creating. The new_bill is going to be made new with the paramater of 5 (which is going to be a $5 bill). new_bill.printing is going to run the object through our new method "printing" and the result is going to be that the console will print out "Made a $5 bill". Cool huh?
